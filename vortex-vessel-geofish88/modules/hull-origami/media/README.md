## 📸 Workbench Telemetry & Visual Capture Protocols

To maintain absolute data integrity and verifiable prior-art across all open-source submissions to the public ledger, all manufacturing snapshots, line traces, or transformation records uploaded here by the community must strictly adhere to these baseline formatting rules:

1.  **High-Contrast Scoring Renders:** All CAD or sheet slicing software vector exports must use a clean, dark-mode background profile to ensure that tool path depths (2.2 mm channels) and remaining hinge web thicknesses are perfectly scannable on workshop monitors.
2.  **Live-Hinge Fatigue Documentation:** High-resolution macro photography demonstrating successful 180-degree elastomeric folding checks must be shot under backlit cross-lighting to visually confirm zero localized cross-link tearing.
3.  **Kinematic Symmetry Logs:** Visual data overlays tracking the concurrent movement of mountain and valley angles during a deployment pull must map directly to the parameters verified inside the root `geofish88-geometry-twin.py` script.
